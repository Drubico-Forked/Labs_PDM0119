Introduccion a Android Studio

Perteneciente a la amplia variadad de aplicaciones de Google y basado en el software IntelliJ IDEA de JetBrains, Android Studio es un Entorno de Desarrollo Integrado (IDE) que proporciona una diversa gama de herramientas y facilidades para el desarrollo de aplicaciones Android destinadas a dispositivos móviles.

Algunas de las características que más resaltan de este IDE son las siguientes (Developer Android):

  • Un sistema de compilación basado en Gradle flexible
  • Un emulador rápido con varias funciones para testing
  • Un entorno unificado en el que puedes realizar desarrollos para todos los dispositivos Android
  • Instant Run para aplicar cambios mientras tu app se ejecuta sin la necesidad de compilar un nuevo APK
  • Integración de plantillas de código y GitHub para ayudarte a compilar funciones comunes de las apps e importar ejemplos de código
  • Gran cantidad de herramientas y frameworks de prueba
  • Herramientas Lint para detectar problemas de rendimiento, usabilidad, compatibilidad de versión, etc.

Creación y configuración de un proyecto básico.

Al inciar Android Studio, si no existe ningún tipo de actualización, error de inicio o algún proyecto ya abierto, se dezpliega la siguiente pantalla, de la cual se seleccionará la opción de Start a New Android Studio project.

Si existe algún proyecto ya abierto dentro del IDE, se tendra que seleccionar desde el menú desplegable de File la opción de New > New Project.

A continuación se desplegará un pantalla donde se elegirá la primera actividad del proyecto; para este laboratorio se seleccionará la actividad vacia.

Luego se configurará el nombre tanto del proyecto como del paquete en el que se trabajará (com.empresa.proyecto); como dato importante, en esta pantalla se seleccionará la API para la cual la aplicación será compatible, que en este caso será la API 16.

Estructura de un proyecto Android.

Una vez creado y configurado el proyecto, se auto generarán muchos ficheros y directorios importantes para el buen funcionamiento del mismo. Conocer la estructura y jerarquia que este posee es muy importante para un desarrollo ágil y efectivo de cualquier aplicación. A continuación se presentará esta estructura y se detallará que función cumple cada uno de los directorios en el proyecto.

  1. manifests: Contiene el archivo AndroidManifest.xml, el cual proporciona información esencial sobre tu aplicación al sistema Android para poder ejecutar el código de la misma.
  2. java: Contiene los archivos fuente .java, en los cuales se desarrolla la lógica de la aplicación.
  3. generatedJava: Contiene todas las clases y librerias auto generadas, las cuales son utilizadas al momento de compilar el proyecto.
  4. res: Contiene todos los ficheros de recursos necesarios para el proyecto: imágenes, layouts, cadenas de texto, etc.
    1. drawable: Contiene las imágenes y otros elementos gráficos usados para la aplicación. A démas se definen diferentes recursos dependiendo de la resolución y densidad de la pantalla.
    2. layout: Contiene los ficheros XML donde se definen las diferentes pantallas de la interfaz gráfica.
    3. mipmap: Contiene los iconos de lanzamiento de la aplicación (el icono que aparecerá en el menú de aplicaciones del dispositivo) para las distintas densidades de pantalla existentes.
    4. values: Contiene otros ficheros XML de recursos de la aplicación, como por ejemplo cadenas de texto (strings.xml), estilos (styles.xml), colores (colors.xml), arrays de valores (arrays.xml), tamaños (dimens.xml), etc.
  5. Gradle Scrips: Contiene los ficheros de configuración del Gradle de Android Studio (Sistema de compliación); estos son auto generados pero pueden ser modificados para diversos usos.

Atajos de teclado para Android Studio

El IDE nos proporciona muchos atajos de teclado, los cuales tienen por fin mejorar la productividad, eficiciencia y eficacia al momento de desarrollar. A continuación se enlistarán algunos atajos que se considerán de mayor uso o más prácticos.

  • Tab Navigation: Estos atajos permiten navegar a través de los editores (tabs) abiertos dentro del IDE. Su uso se resume en la combinación de teclas Alt + Left para la tab izquierda, y Alt + Right para la derecha.
  • Make Project: Compila el proyecto, descargando o actualizando sus dependencias, instanciando recursos, etc. Esta opción esta disponible a través la combinación Shift + F9.
  • Run 'app': Realliza todo el proceso de compilación y busca un dispositivo movil o emulador donde instalar la APK generada; usualmente utilizada para pruebas. Combinación de teclas: Shift + F10.
  • Search Everywhere: Este atajo sirve para buscar coincidencias dentro de todo el proyecto. Se activa presionando dos veces la tecla Shift.
  • Search Class: Este, a diferencia del anterior, solo busca coincidencias con la clases que están dentro del proyecto, y es útil para busquedas más específicas. Su atajo es Ctrl + N.
  • Completion: Provee ayuda de auto completado para atributos, métodos, clases, Macros, sentencias, etc, además mostrará una pequeña descripción de cada item desplegado. Por defecto esta activado al escribir en el editor, pero también se puede hacer uso libre con la combianción de teclas Ctrl + Space.
  • Surround with: Encierra el trozo de código seleccionado en alguna sentencia lógica de encapsulamiento como if, switch, while, for,etc. Para acceder al menú contextual, primero se debe seleccionar alguna porción de código y luego presionar la combinación Ctrl + Alt + T.
  • Generate: Auto genera distintos bloques de códigos dependiendo del contexto en el que se utilice el atajo (Constructores, setters, getters, toString(), etc). Este menú se activa al presionar la combinación Alt + insert.

Sistema de control de versiones (Git).

Recapitulando de cursos anteriores, un control de versiones es la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o la configuración del mismo. Una versión, revisión o edición de un producto, es el estado en el que se encuentra el mismo en un momento dado de su desarrollo o modificación.

El sistema de control de versiones más utilizado es Git. Este es implementado por Android Studio a través de su gama de herraminetas del VCS (del inglés Version Control System); por lo que se permite llevar el control de las versiones del proyecto de forma fácil e intuitiva. Además integra funciones que permiten la conexión con sistemas Git en línea como Github.

Dentro de las facilidades que ofrece el VCS del IDE se encuetran la creación de repositorios, realizar commits, publicar repositorios en línea, realizar pull y push a cualquier repositorio con permisos, etc. A continuación se presentarán los pasos para la publicación de un proyecto, la gestión de versiones, y la clonación de cualquier repositorio en línea.

Creación del repositorio.

En primer lugar se debe seleccionar dentro del menu desplegable VCS, la opción Import Into Version Control > Share Project On Github.

Inmediatamente después el IDE pedirá las credenciales del servicio Github; una vez ingresadas se desplegará la primera pantalla de configuración donde se establecerá el nombre del repositorio y una breve descripción del mismo. Luego se realizará el primer commit, al cual se le podrá agregar un comentario referente al inicio del proyecto. Al finalizar se publicará el repositorio en el sitio de github que se haya especificado.

Manejo de versiones

El manejo básico de versiones en Android Studio se maneja con tres atajos principales commit (Ctrl + k), push (Ctrl + Shift + K) y pull (Ctrl + T).

A continuación se presentará un ejemplo, realizando una modificación a cualquier archivo, generando una nueva versión y publicandola. Al archivo MainActivity.java se le agregará un bloque de comentario; notar que al modificarlo Inmediatamente se muestra como un cambio en la versión actual del VCS.

Clonar repositorio en linea

Otra función que provee el VCS del IDE es la clonación de repositorios publicados en servicios en línea de control de versiones, como Github. Para ello se selecciona, dentro de la pantalla inicial de Android Studio, Check out project from Version Control > Git, e Inmediatamente solicitará el enlace del repositorio a clonar; si se posee los permisos necesarios para clonarlo no se generará ningún problema.

  • Dentro del proyecto realice cualquier cambio a la GUI de la aplicación y realice la publicación de la nueva versión del proyecto.
