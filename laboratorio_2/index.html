Introducción al diseño en Android.

Un aspecto importante dentro del desarrollo móvil es la implementación de una interfaz gráfica con altos estándares de calidad, debido que esta es el primer punto de interacción que tiene el usuario con la aplicación. Android provee una amplia gama de componentes visuales previamente desarrollados y compilados, que facilitan la implementación de cualquier UI.

Andorid realiza el manejo de la UI a través de la declaración de sus componentes en archivos de tipo XML (eXtensible Markup Language); Como todo lenguaje de marcado, XML basa su uso en la utilización de etiquetas, atributos, y valores, permitiendo la declaración de elementos con un alto grado de personalización.

La estructura básica del diseño de una UI en Android se basa en conjuntos de componentes visuales agrupados dentro de contenedores (ViewGroups), los cuales serán inflado por la parte lógica de la app en algún punto de la ejecución.

Ejemplo estructura:

                
        <Viewgroup
            attr = "value"
            attr = "value"
            attr = "value">
            
            <View
                attr = "value"
                attr = "value"
                attr = "value"/>
            
            <View
                attr = "value"
                attr = "value"
                attr = "value"/>
            
            <View
                attr = "value"
                attr = "value"
                attr = "value"/>

        </Viewgroup>

Ejemplo UI Android:

                
        <?xml version="1.0" encoding="utf-8"?>
        <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
            xmlns:tools="http://schemas.android.com/tools"
            android:orientation="vertical"
            android:gravity="center"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            tools:context=".MainActivity">

            <TextView
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:textSize="20sp"
                android:text="Programación de dispositivos móviles."/>
            
            <Button
                android:layout_marginTop="16dp"
                android:padding="24dp"
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:text="Let's Go"/>

        </LinearLayout>

Atributos layout.

Todos los atributos con la sintaxis layout_<something>, modifican el comportamiento de una característica en un View, tomando como referencia el ViewGroup en el que reside.

Componentes básicos de Android UI.

TextView

Referencia Android Developer: TextView.

Componente visual, cuya principal función es mostrar texto al usuario.

Ejemplo:

                    
        <TextView
            android:id="@+id/tv_id"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:textSize="20sp"
            android:text="Programación de dispositivos móviles."/>

EditText

Referencia Android Developer: EditText.

Componente visual, cuya principal función es servir como punto de entrada de texto para el usuario.

Ejemplo:

                    
        <EditText
            android:id="@+id/et_id"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:hint="Programación de dispositivos móviles."
            android:inputType="text"/>

Button

Referencia Android Developer: Button.

Componente visual, cuya principal función es servir como punto de entrada para el usuario, donde se reacciona a partir de la interacción con toques.

Ejemplo:

                    
        <Button
            android:id="@+id/button_send"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="@string/button_send"
            android:onClick="sendMessage"
            android:background="@drawable/button_custom"  />

Referencia Android Developer: CheckBox UI Guide.

Grupos de visuales (View Groups)

Linear Layout

Referencia Android Developer: LinearLayout.

Referencia Android Developer: LinearLayout UI Guide.

Linear Layout es el viewgroup que nos permite posicionar elementos en una dirección, uno después de otro. Se pueden posicionar tanto vertical como horizontalmente.

El principal atributo que nos permite utilizar este layout es layout_weight, el cual permite darle una una proporción de la pantalla, a los elementos hijos con esta propiedad. Lo hace dividiendo el tamaño disponible entre la cantidad de elementos hijos con esta propiedad, y se les asigna el valor que se le dio al atributo. Por ejemplo, si se tiene dos elementos con esta propiedad, con 1 y 2 respectivamente, el tamaño asignado será 1/3 y 2/3 respectivamente. Para usar esta propiedad se debe asignar a layout_width o layout_height cero, dependiendo en que dirección se quiere asignar la proporción.

Ejemplo:

                    
        <LinearLayout
            android:id="@+id/ll_id"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            android:orientation="vertical">

            <EditText
                android:layout_width="match_parent"
                android:layout_height="0dp"
                android:layout_weight="1"
                android:hint="@string/message1" />

            <EditText
                android:layout_width="match_parent"
                android:layout_height="0dp"
                android:layout_weight="1"
                android:hint="@string/message2" />

        </LinearLayout>

Relative Layout

Referencia Android Developer: RelativeLayout.

Referencia Android Developer: RelativeLayout Layout Params.

Referencia Android Developer: RelativeLayout UI Guide.

Este layout nos permite posicionar sus elementos hijos de manera relativa a otro, mediante su id. Las posiciones de estos pueden ser relativas a otros elementos, o relativas a el ViewGroup padre.

Algunos atributos que proporciona relative layout:

  • layout_centerVertical
  • layout_above
  • layout_below
  • layout_toLeftOf
  • layout_toRightOf
  • layout_alignStart
  • layout_alignEnd
  • layout_alignParentTop
  • layout_alignParentBottom

Ejemplo:

                    
        <RelativeLayout
            android:id="@+id/ll_id"
            android:layout_width="match_parent"
            android:layout_height="match_parent">

            <EditText
                android:id="@+id/et_message1"
                android:layout_width="match_parent"
                android:layout_height="wrap_content"
                android:layout_centerVertical="true"
                android:gravity="center"
                android:hint="Mensaje 1" />

            <EditText
                android:id="@+id/et_message2"
                android:layout_width="match_parent"
                android:layout_height="wrap_content"
                android:layout_below="@id/et_message1"
                android:gravity="center"
                android:hint="Mensaje 2" />

            <TextView
                android:id="@+id/tv_id"
                android:layout_width="match_parent"
                android:layout_height="wrap_content"
                android:gravity="center"
                android:layout_above="@id/et_message1"
                android:textSize="20sp"
                android:text="Programación de dispositivos móviles."/>

        </RelativeLayout>

Grid Layout

Referencia Android Developer: GridLayout.

Este ViewGroup dispone de los elementos o nodos hijos adecuándolos, de forma visual, a una cuadrícula predefinida en sus atributos por una cantidad d filas o columnas.

GridLayout se incorporó a la gama de componentes de Android a partir de la API 21; pero con el fin de ser retro compatible, se creó una liberia de compatibilidad para versiones anteriores de esta API. Esta debe de ser declarada en el archivo Gradle:Module en la sección de dependencias.

Implementación de libreria de compatibilidad:

                    
                        implementation 'com.android.support:appcompat-v7:28.0.0'
                        implementation 'com.android.support:gridlayout-v7:28.0.0'

Los atributos más importantes de este componente son columnCount y rowCount, los cuales determinan la cantidad de columnas y filas, respectivamente, y delimitan la cuadrícula del layout. Para los nodos hijos se puede decidir en que celda posicionarlo a través de los atributos layout_column y layout_row

Ejemplo: 

                    
        <android.support.v7.widget.GridLayout 
            xmlns:android="http://schemas.android.com/apk/res/android"
            xmlns:tools="http://schemas.android.com/tools"
            android:id="@+id/GridLayout1"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:columnCount="3"
            tools:context=".GridLayoutActivity" >

            <Button
                android:id="@+id/button3"
                android:layout_column="0"
                android:layout_gravity="left|top"
                android:layout_row="0"
                android:text="Button1" />
            
            <Button
                android:id="@+id/button1"
                android:layout_column="1"
                android:layout_gravity="left|top"
                android:layout_row="0"
                android:text="Button2" />

            <Button
                android:id="@+id/button2"
                android:layout_column="2"
                android:layout_gravity="fill_vertical"
                android:layout_row="0"
                android:layout_rowSpan="2"
                android:text="Button3" />

            <Button
                android:id="@+id/button4"
                android:layout_column="0"
                android:layout_columnSpan="2"
                android:layout_gravity="fill_horizontal"
                android:layout_row="1"
                android:text="Button4" />

        </GridLayout>

Control de eventos.

Referencia Android Developer: Eventos de entrada.

El control de eventos en Android se refiere a la forma en que, tanto la UI como la lógica, deben de comportarse ante interaciones del usuario, pricipalmente representadas por toques en la pantalla.

Estas son capturados a partir de listeners, agregados a las vistas que nos interesa controlar (pricipalmente botones). Los eventos soportados por Android (en modo "táctil") y de los se puede reaccionar ante una interacción del usuario son:

  • onClick(): Se dispara cuando el usuario toca una View en específico.
  • onLongClick(): Se dispara cuando el usuario toca una View en específico y mantiene presionada la pantalla.
  • onFocusChange(): Se dispara cuando el usuario navega a través de la vista o cuando sale de los límites de la misma.
  • onKey(): Se dispara cuando el usuario se centra en una View, y presiona o libera algún botón físico del dispositivo.
  • onTouch(): Se dispara cuando el usuario realiza una acción como un evento táctil, por ejemplo, presionar, liberar o cualquier gesto de movimiento en la pantalla (dentro de los límites del elemento).

A continuación se mostrará la implementación de un listener en una View específica, en este caso un botón con un onClickListener; esto no significa que solo a los botones se le pueden aplicar listeners o que este sea el único listener a implementarse; esto puede realizarse a cualquier objeto que en su cadena de herencia tenga la clase View, y ademas puede implementarse en él cualquiera de los anteriores listeners.

Ejemplo:

Vista:

                
        <Button
            android:id="@+id/button_send"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="@string/button_send"
            android:background="@drawable/button_custom"  />

Lógica:

                
        Button btn_send = findViewById(R.id.button_send);
        btn_send.setOnClickListener(new View.OnClickListener() {
            public void onClick(View v) {
                // Code
            }
        });

Lógica con expresiones lamda:

                
        Button btn_send = findViewById(R.id.button_send);
        btn_send.setOnClickListener(v => {
            // Code
        });

Ejercicios.

  • Realizar una aplicación en Android que simule un "login fake", donde el usuario pueda ingresar un nickname y un password. Debe de existir un botón que al darle click muestre el username en un TextView debajo del botón el username ingresado; si mantiene el click debe de mostrar el password en el mismo TextView.
  • Haciendo uso de GridLayout, realice una galería de imagenes de 3 columnas y 3 filas; al darle click a una imagen cualquiera deberá cambiar de recurso (imagen). Recomendación: Definir una secuencia de imágenes, donde cada imagen tenga un índice que la represente.
  • Haciendo uso de LinearLayout y el diseño con pesos, realizar una UI que contenga 3 paneles distribuidos de forma uniforme con colores predeterminados de fondo (#ff0000, #00ff00, #0000ff). Al hacer click en cualquiera de los paneles este cambiará de color variando únicamente uno de sus componentes RGB (R para el primer panel, y así respectivamente). Recomendación: Guardar los colores en el archivo colors.xml; Al menos 6 colores por panel.
